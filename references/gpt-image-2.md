# GPT Image 2 参考速查

本文件记录 `gpt-image-2` 图片生成/编辑时最常用的官方规则，供 Agent 在需要尺寸、比例、输出格式或排错细节时读取。

## 官方参考

- 图片生成指南: https://platform.openai.com/docs/guides/image-generation
- GPT Image 2 模型页: https://developers.openai.com/api/docs/models/gpt-image-2

## 支持接口

| 场景 | 接口 |
| --- | --- |
| 文生图 | `/v1/images/generations` |
| 改图 / 参考图 / 局部编辑 | `/v1/images/edits` |

## 常见尺寸

`gpt-image-2` 支持 1K、2K、4K 级别尺寸。常用比例建议优先使用下表。

| 比例 | 1K / 常规 | 2K | 4K |
| --- | --- | --- | --- |
| 1:1 方图 | `1024x1024` | `2048x2048` | 不推荐，超出总像素上限 |
| 16:9 横图 | `1536x864` | `2048x1152` | `3840x2160` |
| 9:16 竖图 | `864x1536` | `1152x2048` | `2160x3840` |
| 3:2 横图 | `1536x1024` | `2304x1536` | 可按约束自定义 |
| 2:3 竖图 | `1024x1536` | `1536x2304` | 可按约束自定义 |
| 4:3 横图 | `1280x960` | `2048x1536` | 可按约束自定义 |
| 3:4 竖图 | `960x1280` | `1536x2048` | 可按约束自定义 |

官方热门尺寸包括：

```text
1024x1024
1536x1024
1024x1536
2048x2048
2048x1152
3840x2160
2160x3840
auto
```

## 尺寸约束

自定义 `--size` 必须满足：

- 最大边小于或等于 `3840px`。
- 宽和高都必须是 `16` 的倍数。
- 长边与短边比例不能超过 `3:1`。
- 总像素不少于 `655,360`，不超过 `8,294,400`。
- 超过 `2560x1440` 这类 2K 级别的输出，官方说明目前属于 experimental。

## 质量建议

| 参数 | 建议 |
| --- | --- |
| `low` | 草稿、缩略图、快速迭代 |
| `medium` | 日常成片、成本和质量平衡 |
| `high` | 最终图、海报、复杂细节 |
| `auto` | 让 API 自动选择 |

## 输出格式

| 参数 | 说明 |
| --- | --- |
| `png` | 默认格式，适合保留细节 |
| `jpeg` | 通常更快，适合照片和大图 |
| `webp` | 体积更友好，适合网页 |

`jpeg` 和 `webp` 可以配合 `--output-compression 0-100` 使用。

## 背景与透明

`gpt-image-2` 不支持透明背景。不要传 `background=transparent`，需要背景时使用：

```text
--background auto
--background opaque
```

## 参考图与 mask

- 多参考图使用 Image API 官方字段 `image[]`。
- `gpt-image-2` 会以高保真方式处理输入图片，不要传 `input_fidelity`。
- mask 作用于第一张输入图。
- mask 应与第一张输入图尺寸一致，并带 alpha 通道。
- mask 是提示引导，不保证逐像素严格贴合。

## 提示词建议

生成前尽量明确：

- 主体：人物、产品、场景、物体。
- 构图：特写、半身、全身、俯视、对称、留白。
- 比例：1:1、16:9、9:16，以及对应 `--size`。
- 风格：摄影、插画、海报、国风、商业广告、电影感。
- 光线：柔光、逆光、霓虹、自然光、棚拍。
- 文字：是否需要文字、文字内容、位置和字号。
- 禁止项：不要水印、不要多余文字、不要变形手指、不要额外 logo。
